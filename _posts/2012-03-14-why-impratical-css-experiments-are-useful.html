Lots of people have made lots of amazing things with CSS, using new techniques in HTML5 and CSS3 to push boundaries and explore new possibilities. Many of these demos are incredibly impractical for use in client projects and because of this, have faced criticism from some members of the web community.

Often incredibly unsemantic HTML is the only way to create incredibly beautiful CSS effects. Use of multiple levels of nested divs and countless empty elements promote bad HTML practices. I think the more successful demos are often the ones which manage to do more with less.

Whenever I think up a crazy CSS idea I'd like to try out, I'll always write my optimal HTML first. While restricting, this allows me to create something that is more likely to have real-world uses in the (hopefully not too distant) future. It also means I have to think very carefully about the techniques I use to make sure no line of code is added without good reason. If I need to, I'll allow myself some flexibility — adding a <span> inside an <a> tag is not the end of the world. If I need more than that, I'll often discard the idea as impractical.

An Example

At Clock, we have lots of posters dotted around the office. One that immediately grabbed my attention was 'Shapeset' by Tim Fishlock (ours is not from the original print-run so colours and layout vary slightly). The geometric shapes and bold semi-transparent colours screamed CSS demo so off I went to try it out.

My first step was to write my HTML.

<ul class="alphabet">
  <li class="a">A</li>
  <li class="b">B</li>
  …
</ul>

Seems simple enough, now to start thinking about the CSS. Straight away it's clear that we can rely on border-radius and rgba background colours to do a lot of the work. This means IE8 and lower are out of the window. To create some of the more comlicated letters, such as the C and S, a bit of CSS3 rotation is needed — this works in IE9.

Another important thing to notice is that no letter is made up of more than 3 geomertic shapes. This means we can get away with styling each <li>, along with it's :before and :after pseudo-element to create each letter — no extra markup required.
